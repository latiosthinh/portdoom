<task type="auto">
  <name>Task 1: Create bin/cli.js entry point (per D-03)</name>
  <files>bin/cli.js</files>
  <action>
Create `bin/cli.js` — a CLI script that launches the Electron app and then exits:

1. Create the file with shebang: `#!/usr/bin/env node`
2. Use `child_process.spawn` to launch Electron with the app's main.js:
   - Resolve the electron binary path: `require('electron')` gives the electron module, but for CLI we need the actual electron executable path. Use `path.join(require.resolve('electron'), '..', 'cli.js')` or better, use the `electron` package's CLI path: `require.resolve('electron/cli.js')`
   - Actually, the simpler approach: spawn electron with the app root directory: `spawn('npx', ['electron', '.'], { cwd: path.join(__dirname, '..'), stdio: 'inherit', detached: true })`
   - Better yet: use the electron binary directly: `const electronPath = require('electron');` then spawn it with the app directory
3. The CLI should exit immediately after launching (detached: true, unref the child process)
4. Add error handling: if electron fails to start, log error to stderr and exit with code 1
5. Make the file executable (chmod +x on Unix systems, though this phase is Windows-only)

The CLI behavior per D-03: launches Electron app window, then exits. No persistent terminal process, no headless mode, no subcommands.

Key implementation:
```javascript
#!/usr/bin/env node
const { spawn } = require('child_process');
const path = require('path');
const electron = require('electron');

const appPath = path.join(__dirname, '..');
const child = spawn(electron, [appPath], {
  stdio: 'inherit',
  detached: true,
  env: { ...process.env }
});

child.unref();
// Exit immediately after spawning
process.exit(0);
```

Note: `require('electron')` returns the path to the electron executable when used as a string in spawn.
