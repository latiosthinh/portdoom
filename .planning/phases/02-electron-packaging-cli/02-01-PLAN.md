  <name>Task 2: Update main.js for dynamic port and error handling (per D-01, D-07)</name>
  <files>main.js</files>
  <action>
Modify main.js to support dynamic ports and handle port-in-use errors:

1. **Dynamic port in startServer():** Pass PORT environment variable to the spawned server process:
   - Change the spawn call to include env: `spawn('node', [path.join(__dirname, 'server.js')], { stdio: ['pipe', 'pipe', 'pipe'], windowsHide: true, env: { ...process.env, PORT } })`
   - Define PORT at the top: `const PORT = process.env.PORT || 8765;`

2. **Dynamic port in loadURL():** Change `mainWindow.loadURL('http://localhost:8765')` to `mainWindow.loadURL(\`http://localhost:${PORT}\`)`

3. **Port-in-use error handling (D-07):** Add error handling for when the port is already in use:
   - In startServer(), listen for stderr output that indicates port conflict (e.g., 'EADDRINUSE', 'listen EADDRINUSE', 'address already in use')
   - If port-in-use detected, show Electron dialog: `dialog.showErrorBox('Port Already in Use', \`Port ${PORT} is already in use. Please close the other application or set a different PORT environment variable.\`)`
   - Exit the app gracefully after showing the error: `app.quit()`
   - Import dialog at the top: `const { app, BrowserWindow, Tray, Menu, nativeImage, dialog } = require('electron');`

4. **Server startup timeout:** Add a timeout mechanism (e.g., 5 seconds) so if the server doesn't start within that time, show an error dialog and quit.

The existing main.js structure (createTray, createWindow, app lifecycle handlers) should remain intact. Only the startServer function, loadURL call, and imports need modification.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Electron main → spawned server | Trusted local process, but port could be hijacked if another process binds first |
| BrowserWindow → localhost:PORT | Local-only traffic, no external network exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | server.js spawn | mitigate | Spawn node directly with explicit path, not via shell; validate server.js path exists before spawn |
| T-02-02 | Tampering | PORT env var | accept | PORT is set by user/admin via environment; no external input vector for this desktop app |
| T-02-03 | Information Disclosure | error dialog | mitigate | Error dialog shows only port number and generic message, no system details or stack traces |
</threat_model>
